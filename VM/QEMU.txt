## INSTALLATION QEMU ##
#######################

# To set up the entire stack, you will need to install the packages and start the background virtualization service (libvirtd). Open your terminal and run the command

# 1. Verify Hardware Virtualization

lscpu | grep -i virtualization

# 2. Update the System and Install Packages

sudo pacman -Syu
sudo pacman -S qemu-full libvirt virt-manager virt-viewer edk2-ovmf dnsmasq swtpm guestfs-tools iptables-nft

    # qemu-full: The complete emulator package.
    # libvirt: The API daemon managing your virtual platforms.
    # virt-manager: The GUI application used to create and run VMs.
    # edk2-ovmf: Enables UEFI support for your virtual machines.
    # dnsmasq & iptables-nft: Necessary for default NAT network routing

# 3. Configure User Permissions

sudo gpasswd -a $USER libvirt

# 4. Enable and Start System Services

sudo systemctl enable --now libvirtd.service

# 5. Initialize the Virtual Network

sudo virsh net-autostart default
sudo virsh net-start default


# Q-NOTE (InCase): Vir-Manager Internet connection blocked by UFW

# 1) Change Libvirt's Firewall BackendForcing libvirt to use the iptables backend is the most common fix to prevent UFW from breaking your guest networking.Open the Libvirt network configuration file in a text editor:

sudo nano /etc/libvirt/network.conf

# 2) Search for the firewall_backend line, uncomment it (remove the #), and set it to:

iptables:firewall_backend = "iptables"

# 3) Save the file and restart the libvirt daemon:

sudo systemctl restart libvirtd
